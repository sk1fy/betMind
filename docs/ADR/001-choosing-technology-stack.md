# ADR 001: Выбор технологического стека

## Статус

Accepted

## Дата

2024-01

## Контекст

Необходимо выбрать технологии для:
1. Сбора данных с HLTV
2. Предсказания матчей
3. Пользовательского интерфейса

## Рассматриваемые варианты

### Вариант 1: Python Only

- Скрапер: Selenium/BeautifulSoup
- Алгоритмы: NumPy
- UI: CLI

**Плюсы:** Единый язык  
**Минусы:** Selenium медленный, сложно парсить динамический JS

### Вариант 2: Python + JavaScript (выбран)

- Скрапер: Playwright (JS)
- Алгоритмы: NumPy + Genetic Algorithm
- UI: TUI

**Плюсы:**
- Playwright идеален для HLTV (динамический JS)
- NumPy быстрый для генетического алгоритма
- TUI для терминального интерфейса
- Статистические модели работают без тяжёлого ML

**Минусы:** Два языка

### Вариант 3: Full Python + ML

- Скрапер: Playwright for Python
- ML: NumPy + XGBoost/LightGBM
- UI: Rich/TUI

**Плюсы:** Единый язык  
**Минусы:** Требует много данных для ML, риск переобучения

## Решение

Выбран **Вариант 2** — разделение на:
- **play-scraper**: Node.js + Playwright (сбор данных с HLTV)
- **predictor-v2**: Python + NumPy + Genetic Algorithm (предсказания)

**Ключевые причины:**
- HLTV использует динамический JavaScript → Playwright необходим
- Статистические модели (без ML) работают с малым количеством данных
- Genetic Algorithm для оптимизации 13 параметров
- Цель: 58-65% точности без тяжёлого ML

## Последствия

- Требуется конвертер между проектами
- Разные зависимости (npm vs pip)
- Гибкость в выборе инструментов для каждой задачи
- Ансамбль из 5 статистических моделей
