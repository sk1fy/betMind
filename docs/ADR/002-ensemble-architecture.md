# ADR 002: Архитектура ансамбля моделей

## Статус

Accepted

## Дата

2024-01

## Контекст

Какую архитектуру использовать для предсказания матчей?

## Рассматриваемые варианты

### Вариант 1: Одна ML модель (XGBoost)

- Единая модель с множеством фичей

**Плюсы:** Простота  
**Минусы:** Требует много данных, черный ящик, риск переобучения

### Вариант 2: Ансамбль эвристик (выбран)

- 5 отдельных моделей с разными подходами
- Взвешенное усреднение

**Плюсы:**
- Объяснимые предсказания
- Работает с малым количеством данных
- Легко добавить/удалить модель
- Genetic Algorithm для оптимизации весов

**Минусы:** Сложнее настройка

### Вариант 3: Neural Network

- Глубокое обучение

**Плюсы:** Мощь  
**Минусы:** Нужно 500+ матчей, переобучение, чёрный ящик

## Решение

Выбран **Вариант 2** — Ансамбль из 5 статистических моделей:

| Модель | Вес | Подход |
|--------|-----|--------|
| SimulationModel | 30% | Monte-Carlo симуляция |
| BookmakerModel | 25% | Обратные вероятности от коэффициентов |
| ExpertModel | 20% | Эвристики (форма, карта, пик) |
| MomentumModel | 15% | Тренды, серии, пистолеты |
| ConsensusModel | 10% | Усреднение первых 4 моделей |

**Всего 13 tunable параметров** в EngineParameters, оптимизируемых через Genetic Algorithm.

## Последствия

- Параметры оптимизируются через Genetic Algorithm
- Каждая модель вносит вклад в финальное предсказание
- Возможность отключить модель или изменить вес
- Цель: 58-65% точности без использования ML (XGBoost/LightGBM)
