# ADR 003: Выбор хранилища данных

## Статус

Accepted

## Дата

2024-01

## Контекст

Как хранить исторические данные и предсказания?

## Рассматриваемые варианты

### Вариант 1: SQLite/PostgreSQL

- Полноценная реляционная БД

**Плюсы:** Мощные запросы, индексы  
**Минусы:** Сложность, требует настройки

### Вариант 2: JSON файлы (выбран)

- Человекочитаемые файлы

**Плюсы:**
- Простота
- Не нужна установка БД
- Легко редактировать вручную

**Минусы:**
- Медленные запросы при >10k записей
- Нет индексов

### Вариант 3: DuckDB

- Аналитическая БД

**Плюсы:** Быстрая аналитика  
**Минусы:** Новая технология, меньше документации

## Решение

Выбран **Вариант 2** — JSON файлы:

```
predictor-v2/src/data/
├── historical/
│   └── matches.json       # Исторические матчи для обучения
├── history/
│   └── predictions.json  # История предсказаний
└── configs/
    └── parameters.json   # Оптимизированные параметры
```

## Критерии перехода на SQLite

- >2000 матчей в истории
- Медленные запросы
- Потребность в сложных фильтрах

## Последствия

- Простая архитектура
- Легкий импорт/экспорт
- Ручное редактирование при отладке
- Достаточно для текущего объёма данных
