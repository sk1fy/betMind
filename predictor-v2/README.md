# CS2 Match Predictor v2

Terminal User Interface (TUI) приложение для предсказания исходов матчей Counter-Strike 2. Использует ансамбль ML-моделей и генетический алгоритм для оптимизации.

## Возможности

- **Быстрое предсказание** — ввод только названий команд, рейтингов и коэффициентов
- **Детальное предсказание** — полная статистика команд по каждой карте
- **Генетический алгоритм** — оптимизация весов модели на основе исторических данных
- **Управление историей** — добавление и анализ прошлых матчей
- **Статистика точности** — отслеживание качества предсказаний
- **Экспорт в Markdown** — сохранение предсказаний для дальнейшего использования

## Требования

- Python 3
- NumPy >= 1.24.0

## Установка

```bash
cd predictor-v2
pip install -r requirements.txt
```

## Запуск

```bash
python src/main.py
```

## Меню приложения

1. **New Prediction (Quick)** — быстрое предсказание
2. **New Prediction (Detailed)** — детальное предсказание
3. **Prediction History** — история предсказаний
4. **Optimization (GA)** — запуск генетического алгоритма
5. **Historical Data** — управление историческими данными
6. **Parameters** — настройка параметров модели
7. **Statistics** — статистика точности
8. **Settings** — настройки приложения

## Ансамбль моделей

| Модель | Вес | Описание |
|--------|-----|----------|
| SimulationModel | 30% | Симуляция Монте-Карло |
| BookmakerModel | 25% | Обратные вероятности букмекеров |
| ExpertModel | 20% | Эвристические правила |
| MomentumModel | 15% | Анализ трендов |
| ConsensusModel | 10% | Усреднение моделей |

## Параметры модели

Настраиваемые веса в `src/data/configs/parameters.json`:

- `rank_weight` — влияние рейтинга команды
- `map_weight` — важность статистики на карте
- `pick_bonus` — бонус за пик карты
- `form_multiplier` — множитель текущей формы
- `odds_weight` — вес букмекерских коэффициентов
- И др.

## Интеграция с play-scraper

Данные, собранные `play-scraper`, можно импортировать:
1. Скрапером собрать данные о матчах с HLTV
2. Конвертировать JSON в формат `src/data/historical/matches.json`
3. Запустить оптимизацию для улучшения модели
